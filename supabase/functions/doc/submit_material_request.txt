Edge Function: submit_material_request

Objetivo
- Receber uma requisicao ou devolucao de materiais.
- Aplicar autenticacao e rate limit no endpoint.
- Delegar a regra de estoque para a RPC public.submit_requisicao.

Secrets obrigatorios
- SUPABASE_URL
- SUPABASE_SERVICE_ROLE_KEY

Dependencias no banco
1. 011_create_rate_limit.sql
2. 012_create_project_material_balance.sql
3. 013_update_material_rpcs.sql
4. public.app_users com auth_user_id preenchido

Autenticacao
- Exige Authorization: Bearer <access_token>
- Usa tenant_id do app_users vinculado ao auth.uid()

Rate limit
- Escopo: materials
- Rota: submit_material_request
- Limite atual: 20 chamadas por 60 segundos por tenant + user + device_id

Payload minimo
{
  "client_request_id": "uuid",
  "requisitor": "Nome ou matricula",
  "projeto": "OMI-22-2122",
  "usuario": "almoxarife",
  "data": "2026-03-06T12:00:00Z",
  "tipo_operacao": "REQ",
  "observacao": "",
  "origem": "APP",
  "device_id": "IMEI-OU-ID",
  "itens": [
    {
      "codigo": "210232",
      "descricao": "CAPACITOR...",
      "quantidade": 1,
      "valor_unitario": 0.01
    }
  ]
}

Resposta de sucesso
- HTTP 200
- status APPLIED ou ALREADY_APPLIED em result

Resposta de rejeicao
- HTTP 409
- result.reason pode ser:
  - MATERIAL_NOT_FOUND
  - INSUFFICIENT_STOCK
  - PROJECT_REQUIRED
  - PROJECT_RETURN_EXCEEDS_ISSUED
