Edge Function: sync_run

Objetivo
- Receber do app o resumo de uma sincronizacao executada localmente.
- Aplicar rate limit no servidor para evitar rajadas de chamadas.
- Gravar o resumo em public.sync_runs.
- Opcionalmente gravar etapas em public.sync_run_steps.
- Opcionalmente gravar avisos em public.sync_run_alerts.

Secrets obrigatorios
- SUPABASE_URL
- SUPABASE_SERVICE_ROLE_KEY

Dependencias no banco
1. 009_create_sync_runs.sql
2. 010_create_sync_run_details.sql
3. 011_create_rate_limit.sql
4. Tabela public.app_users com coluna auth_user_id preenchida.

Autenticacao
- Exige Authorization: Bearer <access_token>
- A funcao valida auth.uid() e procura o usuario em public.app_users.

Rate limit
- RPC usada: public.rate_limit_check_and_hit(...)
- Escopo: sync
- Rota: sync.run
- Limite atual: 1 chamada por 30 segundos por tenant + user + device_id

Payload minimo
{
  "sync_uuid": "uuid",
  "device_id": "IMEI-OU-ID",
  "source": "APP",
  "status": "SUCCESS",
  "trigger_type": "MANUAL",
  "started_at": "2026-03-06T12:00:00Z",
  "finished_at": "2026-03-06T12:00:10Z",
  "pending_total": 3,
  "pending_sent": 3,
  "materials_updated": 12,
  "conflicts_found": 1,
  "errors_count": 0,
  "message": "3 pendencias enviadas, 12 itens sincronizados, 1 conflito, 0 erros.",
  "app_version": "1.0-debug"
}

Campos opcionais
- device_label
- network_status
- downloaded_at
- projects_updated
- balances_updated
- warnings_count
- steps: []
- alerts: []

Resposta de sucesso
{
  "success": true,
  "message": "Sincronizacao registrada com sucesso.",
  "sync_uuid": "uuid"
}

Resposta de rate limit
- HTTP 429
- body com retry_after
- header Retry-After

Deploy sugerido
- supabase functions deploy sync_run
- supabase secrets set SUPABASE_URL=... SUPABASE_SERVICE_ROLE_KEY=...
