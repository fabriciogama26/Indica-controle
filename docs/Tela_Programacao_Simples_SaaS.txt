Tela: Programacao SaaS (Novo Cadastro)

Visao geral
-----------
- Tela de cadastro rapido para criar Programacao em lote no mesmo formato da Programacao principal.
- Objetivo: permitir selecionar varias equipes e replicar os mesmos campos do formulario para cada equipe selecionada.
- Fluxo orientado a cadastro: formulario, filtros e lista, sem drag-and-drop.

Arquitetura
-----------
- Rota: `src/app/(dashboard)/programacao-simples/page.tsx`
- View: `src/modules/dashboard/programacao-simples/ProgrammingSimplePageView.tsx`
- Estilo: `src/modules/dashboard/programacao-simples/ProgrammingSimplePageView.module.css`
- API reutilizada: `src/app/api/programacao/route.ts`
- RPC nova:
  - `supabase/migrations/082_create_programming_batch_create_rpc.sql`
  - Funcao: `save_project_programming_batch`
- Migration de permissao:
  - `supabase/migrations/083_add_programacao_simples_page_permissions.sql`
  - `supabase/migrations/084_deactivate_legacy_programacao_page.sql`
- Menu/permissao frontend:
  - `src/components/layout/AppShell.tsx`
  - `src/lib/auth/authorization.ts`
  - `src/modules/dashboard/permissoes/PermissionsPageView.tsx`

Cadastro
--------
- Campos obrigatorios:
  - Projeto
  - Data
  - Periodo
  - Hora inicio
  - Hora termino
  - Equipes (multi-selecao)
- Campos opcionais:
  - Apoio
  - Alimentador
  - Anotacao
  - Documentos (`SGD`, `PI`, `PEP`)
  - Atividades (codigo + quantidade)
- Regra principal:
  - Ao salvar, todos os campos preenchidos no formulario sao replicados para cada equipe selecionada.
  - Cada equipe gera uma programacao independente.
- Persistencia:
  - Frontend envia `POST /api/programacao` com `action = BATCH_CREATE`.
  - Backend chama `save_project_programming_batch`.
  - A RPC reaproveita `save_project_programming` por equipe e faz rollback total se qualquer equipe falhar.

Filtros, auto-complete e listagem
---------------------------------
- Filtros:
  - Data inicial
  - Data final
  - Projeto
  - Equipe
  - Status
- Auto-complete:
  - Atividades via `/api/projects/activity-forecast/catalog`.
- Listagem:
  - Colunas: Data, Projeto, Equipe, Base, Horario, Periodo, Status, Atualizado em.
  - Exportacao CSV da lista filtrada.
  - Paginacao local com `Anterior/Proxima`.

Boas praticas
-------------
- Escopo multi-tenant preservado:
  - API resolve usuario autenticado por tenant.
  - RPC recebe `p_tenant_id` e valida dados via regras existentes.
- Seguranca operacional:
  - Validacao de conflito horario e compatibilidade de base continua centralizada em `save_project_programming`.
  - Em lote, nenhuma insercao parcial permanece se houver erro em uma equipe.
- Auditoria:
  - API registra historico complementar em `app_entity_history` para cada programacao criada no lote (`action = BATCH_CREATE`).

Cancelamento e saldo
--------------------
- Esta tela nao altera saldo de estoque.
- Cancelamento/adiamento continuam no fluxo da tela principal de Programacao.
- O cadastro em lote gera registros `PROGRAMADA`, seguindo as mesmas regras de status da Programacao.

Mapa de Codigo (funcoes, hooks e constantes)
--------------------------------------------
Funcoes principais
- Caminho: `src/modules/dashboard/programacao-simples/ProgrammingSimplePageView.tsx`
  - Responsavel por: formulario, multi-selecao de equipes, filtros, listagem, exportacao e submit em lote.
- Caminho: `src/app/api/programacao/route.ts`
  - Responsavel por: tratar `action = BATCH_CREATE`, validar payload, chamar RPC em lote e registrar historico.
- Caminho: `supabase/migrations/082_create_programming_batch_create_rpc.sql`
  - Responsavel por: transacao de cadastro em lote com rollback total em erro.

Constantes e configuracoes
- `PAGE_SIZE` e `DOCUMENT_KEYS` na tela simples.
- `ROUTE_PAGE_KEYS` em `authorization.ts` com `programacao-simples`.
- `permissionCatalog` em `PermissionsPageView.tsx` com `programacao-simples`.

Funcoes utilitarias relacionadas
- `calculateExpectedMinutes`, `findActivityOption`, `downloadCsvFile`.
- `saveProgrammingBatchViaRpc`, `saveProgrammingBatch`, `registerProgrammingHistory`.

Servicos e integracoes externas
- `save_project_programming_batch` (RPC nova).
- `save_project_programming` (RPC reutilizada por equipe no lote).
- `append_programming_history` (RPC de auditoria funcional).
