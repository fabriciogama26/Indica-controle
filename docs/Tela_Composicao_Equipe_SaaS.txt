Tela: Composicao de Equipe SaaS

Visao geral
-----------
- Tela para registrar a composicao diaria de uma equipe por `Projeto + Equipe + Data`.
- O cadastro parte da equipe selecionada e permite adicionar pessoas do cadastro `Pessoas`.
- A lista exibe uma linha por composicao, mantendo os integrantes dentro dos detalhes/edicao.
- A tela possui dois CSVs:
  - `Exportar lista (CSV)`: exporta uma linha por composicao, com as mesmas colunas exibidas na listagem.
  - `Detalhes (CSV)`: exporta uma linha por integrante, repetindo os dados da composicao em cada linha.

Cadastro
--------
- Campos obrigatorios:
  - Data
  - Projeto
  - Equipe
  - Setor
  - Hora inicial
  - Ao menos uma pessoa
- Campos opcionais:
  - Observacoes
- Ao selecionar uma equipe:
  - A tela exibe placa e encarregado.
  - A tela preenche `Patio` automaticamente com o Centro de Servico vinculado a equipe.
  - O campo `Patio` fica somente leitura quando a equipe possui Centro de Servico.
  - Se a equipe nao possui Centro de Servico, o campo `Patio` pode ser preenchido manualmente e e usado como fallback no salvamento.
  - Se o encarregado existir no cadastro de Pessoas ativo, ele e incluido automaticamente como integrante.
- Inclusao de pessoas:
  - Busca por `Matricula - Nome`.
  - A coluna `Funcao` usa somente o Cargo do colaborador.
  - A coluna `Telefone` repete o telefone do encarregado para todos os integrantes.
  - Bloqueia pessoa duplicada na mesma composicao.
  - Bloqueia matricula duplicada na mesma composicao, mesmo se vier de registros diferentes.
  - Bloqueia composicao com mais de um integrante cuja funcao/cargo contenha `ENCARREGADO`.
  - Permite marcar `Presente = Sim/Nao` por integrante.
- Validacao de salvamento:
  - A tela mostra todos os campos obrigatorios pendentes em uma unica mensagem.
  - Campos verificados: Data, Projeto valido, Equipe valida, Setor, Hora inicial, Patio/Centro de Servico da equipe e ao menos uma pessoa.
  - A API e a RPC `save_team_composition_record` repetem as mesmas regras de campos obrigatorios, duplicidade de matricula e limite de um encarregado.
- Persistencia:
  - Salva snapshots de projeto, equipe, Centro de Servico da equipe em `Patio`, placa, encarregado, pessoa, matricula, CPF, telefone do encarregado e funcao.
  - Mantem o registro historico da composicao mesmo que Pessoas ou Equipes mudem depois.

Boas praticas
-------------
- Toda leitura/escrita usa `tenant_id` do usuario autenticado.
- `team_compositions` e `team_composition_members` possuem RLS por `user_can_access_tenant`.
- O frontend valida projeto por SOB exato antes de filtrar ou salvar.
- A API revalida projeto ativo, equipe ativa e pessoas ativas do tenant antes de salvar.
- Alteracoes relevantes registram entrada em `app_entity_history` com `module_key = composicao-equipe`.

- Ajuste de Patio/Centro de Servico:
  - O valor de `Patio` passa a priorizar o Centro de Servico cadastrado na equipe selecionada.
  - Quando a equipe nao possui Centro de Servico, a API envia `p_yard` para a RPC como fallback.
  - A RPC persiste o snapshot de `Patio` a partir da equipe do tenant ou do fallback `p_yard`.
- Ajuste de regras da composicao:
  - Salvamento passa a listar campos obrigatorios pendentes em vez de retornar mensagem generica.
  - Frontend e API bloqueiam matricula duplicada.
  - Frontend e API bloqueiam mais de um encarregado na mesma composicao.
- Ajuste de persistencia transacional:
  - A gravacao deixou de fazer `insert/update/delete/insert` fragmentado na API.
  - A API passa a chamar `save_team_composition_record`.
  - A RPC valida campos, matricula duplicada, limite de encarregado, projeto/equipe/pessoas do tenant e grava composicao + integrantes na mesma transacao.
  - A RPC so bloqueia concorrencia quando `p_expected_updated_at` e enviado e difere do `updated_at` atual.
  - Em erro de RPC, a API informa se a causa provavel e migration/cache de schema desatualizado.
  - A migration `201_fix_team_composition_save.sql` solicita `notify pgrst, 'reload schema'` apos recriar a RPC.
  - A RPC grava `Funcao` somente com `job_titles.name`, sem concatenar tipo ou nivel do cargo.
  - A RPC grava `Telefone` de todos os integrantes com o telefone do encarregado da equipe.
  - A RPC converte matricula, CPF, telefone, projeto e placa para texto antes de normalizar snapshots.
- Ajuste de listagem:
  - A lista principal passa a exibir uma linha por composicao.
  - Os integrantes ficam disponiveis em Detalhes, Edicao e no CSV detalhado.
  - O CSV `Exportar lista (CSV)` passa a seguir a lista principal, sem uma linha por integrante.
  - O CSV `Detalhes (CSV)` exporta uma linha por integrante.
