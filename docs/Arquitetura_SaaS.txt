Tela: Arquitetura SaaS

Visao geral
-----------
- Documento de referencia da base web criada para o SaaS.
- Cobre stack, rotas iniciais e fronteira com Supabase.

Arquitetura
-----------
- Frontend: Next.js App Router com `src/app` para rotas/layouts e `src/modules` para as telas.
- Hospedagem web: Vercel para publicar o frontend Next.js.
- Backend: Supabase (Edge Functions, Auth, tabelas com RLS e migrations versionadas).
- Login web: `auth-login-web`.
- Login local: `src/app/api/auth/local-login/route.ts`.

Cadastro
--------
- Modulos previstos: Operacao (Projetos, Locacao, Programacao, Medicao), Almoxarifado (Estoque, Entrada e Operacoes de Equipe), Cadastros (Materiais, Pessoas, Cargo, Equipes) e Cadastro Base.

Filtros, auto-complete e listagem
---------------------------------
- Nao se aplica nesta etapa.

Boas praticas
-------------
- `app_users` separado de `people`.
- `login_name` unico globalmente enquanto o tenant nao for resolvido antes do login.
- RLS e tenant scope mantidos no Supabase.
- Edge Functions continuam no Supabase; o Vercel hospeda apenas o app Next e as rotas `src/app/api/*`.
- React Query como base de chamadas ao servidor desde o inicio.

Cancelamento e saldo
--------------------
- Nao se aplica nesta etapa inicial do frontend.

Atalhos
-------
- Rota publica: `/login`
- Rotas protegidas: `/home`, `/projetos`, `/locacao`, `/programacao`, `/materiais`, `/estoque`, `/entrada`, `/saida`, `/pessoas`, `/equipes`, `/tipo-equipe`, `/cadastro-base`
- A rota `/projetos` possui API dedicada em `src/app/api/projects/*` para cadastro e consulta.
