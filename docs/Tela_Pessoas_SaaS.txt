Tela: Pessoas SaaS

Visao geral
-----------
- Tela para cadastro e manutencao de pessoas operacionais por tenant.
- Permite cadastrar, editar, consultar, cancelar/ativar e auditar alteracoes.
- Consome catalogos de `Cargo`, `Tipo por Cargo` e `Nivel`.
- Segue o mesmo padrao visual de `Projetos`, `Atividades` e `Equipes`.

Arquitetura
-----------
- Rota: `src/app/(dashboard)/pessoas/page.tsx`
- View: `src/modules/dashboard/pessoas/PeoplePageView.tsx`
- Estilo: `src/modules/dashboard/pessoas/PeoplePageView.module.css`
- API principal: `src/app/api/people/route.ts`
- API de metadados: `src/app/api/people/meta/route.ts`
- Banco (base people/job_titles): `supabase/migrations/014_create_people_job_titles.sql`
- Banco (matricula): `supabase/migrations/035_add_matriculation_to_people.sql`
- Banco (tipos por cargo): `supabase/migrations/047_create_job_title_types_and_people_type_link.sql`
- Banco (niveis): `supabase/migrations/048_create_job_levels_and_people_level_link.sql`
- Banco (historico generico): `supabase/migrations/051_create_app_entity_history_and_activity_status.sql`
- Banco (status/cancelamento people): `supabase/migrations/055_add_people_status_and_cancellation_fields.sql`
- Banco (bloqueio de duplicidade people): `supabase/migrations/056_prevent_people_duplicate_identity.sql`

Cadastro
--------
- Campos obrigatorios:
  - `Nome`
  - `Cargo`
- Campos opcionais:
  - `Matricula`
  - `Tipo` (dependente do cargo selecionado)
  - `Nivel`
- Regras:
  - `Tipo` so lista opcoes vinculadas ao `Cargo` selecionado.
  - `Nivel` precisa existir em `job_levels` ativo no tenant.
  - `Tipo` precisa existir em `job_title_types` ativo no tenant e no mesmo cargo.
  - Bloqueio de duplicidade por combinacao: `Nome + Matricula + Cargo + Tipo + Nivel` no mesmo tenant.
  - Edicao de registro inativo e bloqueada ate reativacao.
- Fluxos:
  - `POST /api/people` para cadastro.
  - `PUT /api/people` para edicao.
  - `PATCH /api/people` para cancelamento/ativacao com motivo obrigatorio.

Filtros, auto-complete e listagem
---------------------------------
- Filtros:
  - `Nome`
  - `Matricula`
  - `Cargo`
  - `Tipo` (dependente de cargo no filtro)
  - `Nivel`
  - `Status`
- Listagem:
  - Paginacao server-side (`page`, `pageSize`, `total`)
  - Colunas: Nome, Matricula, Cargo, Tipo, Nivel, Registrado em, Acoes
  - Acoes: `Detalhes`, `Editar`, `Historico`, `Cancelar/Ativar`
  - Botao `Exportar Excel (CSV)` no cabecalho
- Historico:
  - Fonte: `app_entity_history`
  - Modal com paginacao fixa de `5` registros por pagina

Boas praticas
-------------
- Sempre validar `tenant_id` em todas as operacoes.
- Nao aceitar `Tipo` fora do `Cargo` escolhido.
- Nao aceitar `Nivel` inexistente/inativo no tenant.
- Registrar alteracoes em `app_entity_history` para `UPDATE`, `CANCEL`, `ACTIVATE`.
- Manter motivo obrigatorio em cancelamento/ativacao.

Cancelamento e saldo
--------------------
- Cancelamento/ativacao implementado em `people`.
- Campos de status:
  - `ativo`
  - `cancellation_reason`
  - `canceled_at`
  - `canceled_by`
- Regra de consistencia:
  - Ativo: sem dados de cancelamento.
  - Inativo: com data e motivo.
- Saldo:
  - Nao se aplica ao modulo de pessoas.
